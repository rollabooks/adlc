# Chapter 4 — Project 1: Hello World in 0-Code

## What You'll Build

An interactive Python program that:
- Greets the user by name
- Calculates their age from their birth year
- Generates a random "fortune of the day"
- Has working automated tests

**Estimated time**: 15–20 minutes

---

## 4.1 — Setting Up the Project (ADLC Phases 0–2)

### 🔧 Hands-on — Initial Setup

1. Open VS Code
2. Create a new folder: `hello-0code`
3. Open the folder in VS Code (`File → Open Folder`)
4. Create a `_CONTEXT.md` file in the root with the following content:

```markdown
# Project: Hello 0-Code

## Purpose
An interactive Python program that greets the user, calculates their age,
and generates a random "fortune of the day."

## Technologies
- Python 3.11+
- Standard library only (random, datetime)
- Testing with pytest

## Structure
hello-0code/
├── _CONTEXT.md
├── main.py          ← Entry point
├── greeter.py       ← Greeting and calculation logic
├── fortunes.py      ← Fortune list and generator
└── tests/
    ├── test_greeter.py
    └── test_fortunes.py

## Conventions
- snake_case for everything
- Type hints on all functions
- Docstrings for every public function
- Formatted output with f-strings

## Constraints
- Standard Python libraries only. No pip install.
- Birth year must be validated (1900–current year)
- Name cannot be empty
- Handle invalid input with clear messages
```

### 🎯 CHECKPOINT
You should have a folder with a single file: `_CONTEXT.md`. Everything else will be generated by the AI.

## 4.2 — The First Prompt (ADLC Phases 3–4)

### 🔧 Hands-on — Generating the Code

1. Open Copilot Chat in Agent Mode (`Ctrl+Shift+I`, select "Agent")

2. Type this prompt:

```text
Read the _CONTEXT.md file and implement the entire project following
the structure and constraints defined there.

The program should:
1. Ask the user for their name
2. Ask for their birth year
3. Calculate their age
4. Display a personalized greeting with their age
5. Display a random fortune of the day (at least 10 different fortunes)

Also create unit tests for greeter.py and fortunes.py.
```

3. **Watch the AI at work**: Copilot in Agent Mode will create files one by one. For each file, it will show you the content and ask whether to accept the changes.

4. **Accept or correct**: If the code looks right, accept it. If you notice anything off, ask for a change.

## 4.3 — Analyzing the Output

Let's look at what the AI should have generated. Don't worry if your output is slightly different — what matters is that it follows the structure defined in `_CONTEXT.md`.

### Expected output: `greeter.py`

```python
from datetime import datetime


def get_user_name() -> str:
    """Chiede il nome all'utente e lo valida."""
    while True:
        name = input("Come ti chiami? ").strip()
        if name:
            return name
        print("Il nome non può essere vuoto. Riprova.")


def get_birth_year() -> int:
    """Chiede l'anno di nascita e lo valida."""
    current_year = datetime.now().year
    while True:
        try:
            year = int(input("In che anno sei nato/a? "))
            if 1900 <= year <= current_year:
                return year
            print(f"L'anno deve essere tra 1900 e {current_year}.")
        except ValueError:
            print("Inserisci un numero valido.")


def calculate_age(birth_year: int) -> int:
    """Calcola l'età a partire dall'anno di nascita."""
    return datetime.now().year - birth_year


def greet(name: str, age: int) -> str:
    """Genera il messaggio di saluto personalizzato."""
    return f"Ciao {name}! Hai {age} anni."
```

### What to Look For

Check that the AI followed the `_CONTEXT.md`:

| Context constraint | Met? |
|:--|:--|
| snake_case for functions | ✅ `get_user_name`, `calculate_age` |
| Type hints | ✅ `-> str`, `-> int` |
| Docstrings | ✅ On every function |
| Year validation (1900–current) | ✅ `if 1900 <= year <= current_year` |
| Name not empty | ✅ `if name:` |
| Standard library only | ✅ Only `datetime` |

> 💡 **Tip**: This kind of verification is your main job as a 0-code developer. You don't have to write the code, but you do need to know how to **read and validate** that it meets the requirements.

## 4.5 — Iterating and Improving

The real power of 0-code shines during iteration. Let's try adding a feature without touching the code ourselves.

### 🔧 Hands-on — Adding a Feature

In the Copilot chat, type:

```text
Aggiungi una funzionalità "zodiac": dopo il saluto, mostra il segno zodiacale 
dell'utente basato sull'anno di nascita (zodiaco cinese). 
Aggiungi la logica in un nuovo file zodiac.py con i relativi test.
Aggiorna main.py per usare questa nuova funzionalità.
```

The AI will:
1. Create `zodiac.py` with the Chinese zodiac logic
2. Create `tests/test_zodiac.py`
3. Modify `main.py` to integrate the new feature
4. Automatically follow the conventions from `_CONTEXT.md` (snake_case, type hints, docstrings)

### 🔧 Hands-on — Fixing a Bug

If the program behaves unexpectedly, don't edit the code yourself. Describe the problem to Copilot:

```text
Quando inserisco l'anno 2024 come anno di nascita, il programma dice 
che ho 2 anni. Dovrebbe dire 1 anno se non ho ancora compiuto gli anni 
quest'anno, o 2 se li ho già compiuti. Correggi calcolando con mese e 
giorno, non solo l'anno.
```

The AI will fix the code and update the related tests.

## 4.6 — Lessons from the First Project (ADLC Phase 6)

### What You Learned

1. **`_CONTEXT.md` drives everything**: file structure, conventions, constraints — the AI followed them because you defined them in the context.

2. **The initial prompt can be generic**: you don't need to spell out every detail in the prompt. The details live in the context; the prompt states the goal.

3. **Iteration is free**: adding features or fixing bugs is as fast as describing the problem.

4. **You review, the AI implements**: your job was to read the code and verify it met the requirements, not to write it.

### 🔧 Hands-on — Update the Context

If the AI made recurring mistakes during this project, add them as constraints to `_CONTEXT.md`. For example:

```markdown
## Lessons Learned
- Calculate age using a full datetime.date, not just the year
- Tests should use fixtures for test data, not hardcoded values
```

This context will serve as the foundation for your future projects.
